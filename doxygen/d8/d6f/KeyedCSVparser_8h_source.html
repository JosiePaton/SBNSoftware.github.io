39 /**
40  * @brief A set of key/values items.
41  *
42  * This class collects `Item` objects, each being a string key and a sequence
43  * of zero or more values. An specific item can be accessed by its key
44  * (`findItem()`, `getItem()`) or all items may be iterated through (`items()`).
45  *
46  *
47  * Initialization and updates
48  * ---------------------------
49  *
50  * A `KeyValuesData` object always starts empty (default constructor).
51  * A new item is also always created empty (`makeItem()`) and with a set key.
52  *
53  * After an empty item (i.e. an item with a key but no values) is created,
54  * values can be added using the `Item` subclass interface (`addValue()`).
55  * Item values and keys can be modified by changing `Item` data members
56  * directly; note however that the mean to do that is the reference that
57  * the `makeItem()` call returned, as there is no other way to access an item
58  * in a writeable way.
59  *
60  * This object will refuse to create a new item with the same key as an existing
61  * one. However, the key of the item may be changed to any value after
62  * `makeItem()` is called, although the interface does not encourage that.
63  * If this introduces a duplicate key, the query functions will systematically
64  * retrieve only one of the items with the repeated key (which one and whether
65  * always the same one are undefined).
66  *
67  * Example:
68  * ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~{.cpp}
69  * icarus::details::KeyValuesData data;
70  * data.makeItem("TriggerType").addValue("S5");
71  * data.makeItem("Triggers");
72  * data.makeItem("TriggerWindows").addValue("0C0B");
73  * data.makeItem("TPChits")
74  * .addValue("12").addValue("130").addValue("0").addValue("0");
75  * data.makeItem("TPChitTimes")
76  * .addValue("3").addValue("-1.1").addValue("-0.3").addValue("0.1");
77  * data.makeItem("PMThits").addValue("8");
78  * ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
79  * creates a `data` object with four items, a `TriggerType` one with one value,
80  * a `Triggers` one with no values, a `TriggerWindows` with a single value
81  * (expressed as a hexadecimal number), a `TPChits` one with four values,
82  * a `TPChitTimes` with four values (the first meant to be the number of
83  * remaining ones) and a `PMThits` with one.
84  *
85  *
86  * Query
87  * ------
88  *
89  * The interface is quite terse.
90  *
91  * General query methods reports whether there is no item in the object
92  * (`empty()`) and how many items there are (`size()`).
93  *
94  * A item with a known key can be retrieved (`getItem()`, `findItem()`), or
95  * its existence may be tested (`hasItem()`).
96  *
97  * Finally, all items can be iterated (`items()`). In this case, the items
98  * are presented in the creation order.
99  *
100  * The `Item` interface is documented in that subclass.
101  *
102  * Example using the `data` object from the previous example:
103  * ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~{.cpp}
104  * std::string triggerType = data.getItem("TriggerType").values()[0];
105  * std::vector<int> triggers = data.getItem("Triggers").getVector<int>();
106  * std::uint32_t triggerWindowBits
107  * = data.getItem("TriggerWindows").getNumber<std::uint32_t>(0, 16); // base 16
108  * std::vector<int> TPChits = data.getItem("TPChits").getVector<int>();
109  * std::vector<float> TPCtimes
110  * = data.getItem("TPChitTimes").getSizedVector<float>();
111  * std::vector<int> CRThits;
112  * if (auto const* item = data.findItem("CRThits"))
113  * CRThits = item->getVector<int>();
114  * ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
115  *
116  */

382 /**
383  * @brief Parser to fill a `KeyValuesData` structure out of a character buffer.
384  *
385  * It currently supports only single-line buffer.
386  *
387  * The parser operates one "line" at a time, returning a `KeyValuesData` with
388  * the values assigned to each detected key. No data type is implied: all
389  * elements are treated as strings, either a key or a value.
390  * The parser separates the elements according to a separator, strips them of
391  * trailing and heading spaces, then it decides whether each element is a value
392  * to be assigned to the last key found, or a new key.
393  * Keys are elements that have letters in them, values are anything else.
394  * This simple (and arguable) criterion can be broken with specific parser
395  * configuration: a pattern can be specified that when matched to an element
396  * will make it a key; the pattern can also set the number of values that key
397  * will require.
398  *
399  * For example:
400  * ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~{.cpp}
401  * icarus::details::KeyedCSVparser parser;
402  * parser.addPatterns({
403  * { "TriggerType", 1U } // expect one value (even if contains letters)
404  * , { "TriggerWindows", 1U } // expect one value (even if contains letters)
405  * , { "TPChitTimes", icarus::details::KeyedCSVparser::FixedSize }
406  * // the first value is an integer, count of how many other values
407  * });
408  *
409  * icarus::details::KeyValuesData data = parser(
410  * "TriggerType, S5, Triggers, TriggerWindows, 0C0B,"
411  * " TPChits, 12, 130, 0, 0, TPChitTimes, 3, -1.1, -0.3, 0.1, PMThits, 8"
412  * );
413  * ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
414  * will return `data` with 6 items.
415  */

458  /**
459  * @name Know patterns
460  *
461  * The parser normally treats as a value everything that does not start with a
462  * letter.
463  * Known patterns may override this behaviour: if a token matches a known
464  * pattern, it is considered a key and it is possible to specify the expected
465  * number of values.
466  *
467  * The number of values can be:
468  * * a number: exactly that number of values are required; an exception will
469  * be thrown if not enough tokens are available;
470  * * `FixedSize`: the next token must be a non-negative integer specifying how
471  * many other values to add (read this with `Item::getSizedVector()`);

473  * * `DynamicSize`: the standard algorithm is used and values are added as
474  * long as they don't look like keys; the token matching the pattern is
475  * interpreted as a key though.

477  * Patterns are considered in the order they were added.
