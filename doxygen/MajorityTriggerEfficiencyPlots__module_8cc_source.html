87 /**
88  * @brief Produces plots about trigger simulation and trigger efficiency.
89  *
90  * This module is an implementation of `TriggerEfficiencyPlotsBase`
91  * for a trigger defined as a minimum number of trigger primitives beyond
92  * threshold.
93  *
94  * A trigger primitive is a two-level function of time which describes when
95  * that primitive is on and when it is off. Trigger primitives are given as
96  * input to this module and their origin may vary, but the standard source in
97  * ICARUS is the pairing with AND or OR of two optical detector channels
98  * discriminated against a certain ADC count threshold.
99  *
100  *
101  * Trigger logic algorithm
102  * ========================
103  *
104  * @anchor MajorityTriggerEfficiencyPlots_Algorithm
105  *
106  * This section describes the trigger logic algorithm used in
107  * `icarus::trigger::MajorityTriggerEfficiencyPlots` and its assumptions.
108  *
109  * The algorithm keeps the trigger primitives from the different cryostats
110  * separate for the most time.
111  * Within each cryostat, all trigger primitives are treated equally, whether
112  * they originate from one or from two channels (or 10 or 30), and wherever
113  * their channels are in the cryostat.
114  * The trigger primitives in each cryostat are combined in a multi-level gate by
115  * adding them, so that the level of the resulting gate matches at any time how
116  * many trigger primitives are on at that time.
117  * Finally, the maximum number of trigger primitives open in any of the
118  * cryostats at each time is the level to be compared to the trigger
119  * requirements.
120  *
121  * This multi-level gate is set in coincidence with the beam gate by multiplying
122  * the multi-level and the beam gates.
123  * The beam gate opens at a time configured in `DetectorClocks` service provider
124  * (`detinfo::DetectorClocks::BeamGateTime()`) and has a duration configured
125  * in this module (`BeamGateDuration`).
126  *
127  * At this point, the trigger gate is a multi-level gate suppressed everywhere
128  * except than during the beam gate.
129  * The algorithm handles multiple trigger primitive requirements.
130  * Each requirement is simply how many trigger primitives must be open at the
131  * same time in a single cryostat for the trigger to fire. The values of these
132  * requirements are set in the configuration (`MinimumPrimitives`).
133  * To determine whether a trigger with a given requirement, i.e. with a required
134  * minimum number of trigger primitives open at the same time, has fired, the
135  * gate combined as described above is scanned to find _the first tick_ where
136  * the level of the gate reaches or passes this minimum required number. If such
137  * tick exists, the trigger is considered to have fired, and at that time.
138  *
139  * As a consequence, there are for each event as many different trigger
140  * responses as how many different requirements are configured in
141  * `MinimumPrimitives`, _times_ how many ADC thresholds are provided in input,
142  * configured in `Thresholds`.
143  *
144  * While there _is_ a parameter describing the time resolution of the trigger
145  * (`TriggerTimeResolution`), this is currently only used for aesthetic purposes
146  * to choose the binning of some plots: the resolution is _not_ superimposed
147  * to the gates (yet).
148  *
149  * The combination algorithm is implemented in `combineTriggerPrimitives()`
150  * while the requirement evaluation and plotting are implemented in
151  * `plotResponses()`.
152  *
153  * The set of plots and their organization are described in the documentation of
154  * `icarus::trigger::TriggerEfficiencyPlotsBase`.
155  * In the following documentation only the additions are described.
156  *
157  *
158  * Output plots
159  * -------------
160  *
161  * A generic "setting" of `icarus::trigger::TriggerEfficiencyPlotsBase` is
162  * in this module represented by the single trigger primitive requirement _N_
163  * (configuration parameter: `MinimumPrimitives`). The folders and plots will
164  * identify each requirement with the tag `ReqN` (e.g. `Req5` when requesting
165  * at least 5 trigger primitives for an event trigger).
166  *
167  * There are a few plots that are produced by this module in addition to the
168  * ones in `TriggerEfficiencyPlotsBase`.
169  *
170  * There are different "types" of plots. Some
171  * @ref MajorityTriggerEfficiencyPlots_SelectionPlots "do not depend on triggering at all",
172  * like the deposited energy distribution. Others
173  * @ref MajorityTriggerEfficiencyPlots_MultiTriggerPlots "cross different trigger definitions",
174  * like the trigger efficiency as function of trigger requirement. Others still
175  * @ref MajorityTriggerEfficiencyPlots_SingleTriggerPlots "assume a single trigger definition":
176  * this is the case of trigger efficiency plots versus energy. Finally, there are
177  * @ref MajorityTriggerEfficiencyPlots_SingleTriggerResponsePlots "plots that depend on a specific trigger definition and outcome":
178  * this is the case of all the plots including only triggering or non-triggering
179  * events.
180  *
181  * A list of additional plots follows for each plot type.
182  * All the plots are always relative to a specific optical detector channel
183  * threshold (ADC) and a broad event category.
184  *
185  *
186  * ### Plots independent of the triggers (selection plots)
187  *
188  * @anchor MajorityTriggerEfficiencyPlots_SelectionPlots
189  *
190  * Only @ref TriggerEfficiencyPlotsBase_SelectionPlots "the standard plots"
191  * from `TriggerEfficiencyPlotsBase` are produced in this category.
192  *
193  *
194  * ### Plots including different trigger requirements
195  *
196  * @anchor MajorityTriggerEfficiencyPlots_MultiTriggerPlots
197  *
198  * In addition to @ref TriggerEfficiencyPlotsBase_MultiTriggerPlots "the plots"
199  * from `TriggerEfficiencyPlotsBase`, the following plots are also produced:
200  *
201  * * `Eff`: trigger efficiency defined as number of triggered events over the
202  * total number of events, as function of the minimum number of trigger
203  * primitives (`MinimumPrimitives`) to define a firing trigger; uncertainties
204  * are managed by `TEfficiency`.
205  * * `TriggerTick`: distribution of the time of the earliest trigger for the
206  * event, as function of the minimum number of trigger primitives (as in
207  * `Eff`). It may happen that the event is such that there is e.g. a
208  * 20-primitive flash, then subsiding, and then another 30-primitive flash.
209  * In such a case, in the trigger requirement "&geq; 15 primitives" such event
210  * will show at the time of the 20-primitive flash, while in the trigger
211  * requirement "&geq; 25 primitives" it will show at the time of the
212  * 30-primitive flash. Each event appears at most once for each trigger
213  * requirement, and it may not appear at all if does not fire a trigger.
214  * * `NPrimitives`: the maximum number of primitives "on" at any time.
215  *
216  *
217  * ### Plots depending on a specific trigger definition
218  *
219  * @anchor MajorityTriggerEfficiencyPlots_SingleTriggerPlots
220  *
221  * Only @ref TriggerEfficiencyPlotsBase_SingleTriggerPlots "the standard plots"
222  * from `TriggerEfficiencyPlotsBase` are produced in this category.
223  *
224  *
225  * ### Plots depending on a specific trigger definition and response
226  *
227  * @anchor MajorityTriggerEfficiencyPlots_SingleTriggerResponsePlots
228  *
229  * Only @ref TriggerEfficiencyPlotsBase_SingleTriggerResponsePlots "the standard plots"
230  * from `TriggerEfficiencyPlotsBase` are produced in this category.
231  *
232  *
233  * Configuration parameters
234  * =========================
235  *
236  * @anchor MajorityTriggerEfficiencyPlots_Configuration
237  *
238  * In addition to
239  * @ref TriggerEfficiencyPlotsBase_Configuration "all the configuration parameters"
240  * from `TriggerEfficiencyPlotsBase`, the following one is also present:
241  *
242  * * `MinimumPrimitives` (list of integers, _mandatory_): a list of alternative
243  * requirements for the definition of a trigger; each value is the number
244  * of trigger primitives needed to be "on" at the same time for the trigger
245  * to fire;
246  *
247  * An example job configuration is provided as `maketriggerplots_icarus.fcl`.
248  *
249  *
250  * Technical description of the module
251  * ====================================
252  *
253  * This module class is derived from
254  * `icarus::trigger::TriggerEfficiencyPlotsBase`, which provides a backbone to
255  * perform the simulation of triggers and plotting of their efficiency.
256  *
257  * There is not any superior design involved in this separation, but just the
258  * desire to share most code possible between different modules which simulate
259  * different trigger patterns and as a consequence might have specific plots to
260  * fill.
261  *
262  * This module redefines:
263  *
264  * * `initializePlotSet()` to define the
265  * @ref MajorityTriggerEfficiencyPlots_MultiTriggerPlots "few additional plots"
266  * needed;
267  * * `simulateAndPlot()`, which must always be defined, and which connects
268  * the simulation pieces and the plotting.
269  *
270  * It does not redefine `initializeEfficiencyPerTriggerPlots()` nor
271  * `initializeEventPlots()` because there are no additional plots of the types
272  * these functions deal with.
273  *
274  * The event categories are from the default list (`DefaultPlotCategories`) too.
275  *
276  */

393  /**
394  * @brief Completes the event trigger simulation and fills the plots.
395  * @param iThr index of PMT threshold (used in tree output)
396  * @param threshold PMT threshold in ADC counts (for printing)
397  * @param plotSets set of plot boxes to fill (from `initializePlotSet()`)
398  * @param eventInfo event information for plotting
399  * @param combinedCounts combined trigger primitives, per cryostat
400  *
401  * For each of the trigger requirements (`MinimumPrimitives`), this method:
402  *
403  * 1. applies the requirement to the `combinedCounts` trigger primitive
404  * 2. computes the event triggers (including the main one)
405  * 3. fills all plots in all the plot sets for this requirement accordingly
406  *
407  * The input combined trigger primitives contain the maximum number of
408  * trigger primitives active at each optical clock tick, one entry for each
409  * cryostat.
410  * It is assumed that the beam gate has already been "applied" so that outside
411  * it no trigger primitive is considered open.
412  *
413  * A trigger with requirement of minimum trigger primitives _N_ is fired if
414  * this combined primitive reaches or passes _N_, i.e. if there are at least
415  * _N_ open trigger primitives at any time.
416  * The time of the trigger is taken as the first tick at which the requirement
417  * is met.
418  *
419  * Extra plots are an overview of the trigger efficiency for different
420  * requirements, per threshold, the distribution of the trigger time for
421  * different requirements, and maximum number of open primitive in a cryostat,
422  * per event.
423  *
424  * Note that there is no information about which cryostat is meeting the
425  * trigger requirement.
426  */

710 //------------------------------------------------------------------------------
711 void icarus::trigger::MajorityTriggerEfficiencyPlots::plotResponses(
712  std::size_t iThr,
713  std::string const& threshold,
714  PlotSandboxRefs_t const& plotSets,
715  EventInfo_t const& eventInfo,
716  detinfo::DetectorClocksData const& clockData,
717  std::vector<TriggerGateData_t> const& combinedCounts,
718  std::vector<ChannelID_t> const& channelList
719 ) {
720 
721  /*
722  * This function plots according to the configured minimum number of trigger
723  * primitives: for each requirement of minimum number of primitives, the
724  * earliest time where that requirement is met is found, and that is
725  * considered as the trigger time.
726  *
727  * The following quantities are drawn per ADC threshold and per plot category:
728  *
729  * * per minimum number of primitives:
730  * * trigger efficiency (i.e. whether there was _any time_ a number of
731  * primitives fulfilling the requirement)
732  * * trigger time in ticks (distribution as 2D histogram)
733  * * maximum number of trigger primitives present at any time
734  * * deposited energy during beam spill
735  *
736  */
737  using namespace std::string_literals;
738 
739  using ClockTick_t = TriggerGateData_t::ClockTick_t;
740  using OpeningCount_t = TriggerGateData_t::OpeningCount_t;
741 
742  using PrimitiveCount_t = std::pair<ClockTick_t, OpeningCount_t>;
743 
744  PrimitiveCount_t maxPrimitives { ClockTick_t{ 0 } /* dummy */, 0U };
745  for (auto const& [ iCryo, combinedCount ]: util::enumerate(combinedCounts)) {
746  auto const maxPrimitiveTime { combinedCount.findMaxOpen() };
747  PrimitiveCount_t const maxPrimitivesInCryo
748  { maxPrimitiveTime, combinedCount.openingCount(maxPrimitiveTime) };
749  if (maxPrimitivesInCryo.second > maxPrimitives.second)
750  maxPrimitives = maxPrimitivesInCryo;
751 
752  mf::LogTrace log { helper().logCategory() };
753  log << "Max primitive count in " << threshold << " for C:" << iCryo << ": "
754  << maxPrimitivesInCryo.second;
755  if (maxPrimitivesInCryo.second > 0) {
756  log << " at tick " << maxPrimitivesInCryo.first << " ("
757  << detinfo::DetectorTimings(clockData).toElectronicsTime
758  (detinfo::DetectorTimings::optical_tick{ maxPrimitivesInCryo.first })
759  << ")"
760  ;
761  } // if
762  } // for
763 
764  PMTInfo_t const PMTinfo { threshold, channelList };
765 
766  /*
767  * Fill all the histograms for all the minimum primitive requirements
768  * (filling the information whether or not the trigger fired),
769  * for all the qualifying categories.
770  * Note that in this type of plots each event appears in all bins
771  * (may be with "fired" or "not fired" on each bin)
772  */
773 
774  // PrimitiveCount_t lastMinCount { TriggerGateData_t::MinTick, 0 };
775 
776  bool fired = true; // the final trigger response (changes with requirement)
777 
778  for (auto [ iReq, minCount ]: util::enumerate(fMinimumPrimitives)) {
779 
780  TriggerInfo_t triggerInfo;
781  if (fired) { // this is still the previous requirement
782  for (auto const& [ iCryo, cryoGate ]: util::enumerate(combinedCounts)) {
783 
784  icarus::trigger::details::GateOpeningInfoExtractor extractOpeningInfo
785  { cryoGate, minCount };
786  extractOpeningInfo.setLocation(iCryo);
787  while (extractOpeningInfo) {
788  auto info = extractOpeningInfo();
789  if (info) triggerInfo.add(info.value());
790  } // while
791  } // for all cryostats
792  triggerInfo.sortOpenings(); // sort in time
793  fired = triggerInfo.fired();
794  } // if previous fired
795 
796  if (fired) {
797  mf::LogTrace log(helper().logCategory());
798  log
799  << " => fired (>" << minCount << ") at " << triggerInfo.atTick()
800  << " (level " << triggerInfo.level() << ") from"
801  ;
802  if (triggerInfo.hasLocation()) log << " C:" << triggerInfo.location();
803  else log << " unknown location";
804  log << ", " << triggerInfo.nTriggers() << " triggers total:";
805  for (auto const& [iTrigger, info ]: util::enumerate(triggerInfo.all())) {
806  log << " [" << iTrigger << "] at " << info.tick;
807  if (info.hasLocation()) log << " of C:" << info.locationID;
808  else log << " [unknown location]";
809  log << " (level=" << info.level << ")";
810  } // for
811 
812  } // if fired
813  else {
814  mf::LogTrace(helper().logCategory())
815  << " => not fired (>" << minCount << ")";
816  }
817 
818  // at this point we know we have minCount or more trigger primitives,
819  // and the time of this one is in lastMinCount.first (just in case)
820 
821  if (fResponseTree) fResponseTree->assignResponse(iThr, iReq, fired);
822 
823  registerTriggerResult(iThr, iReq, fired);
824 
825  std::string const minCountStr { "Req" + std::to_string(minCount) };
826 
827  // go through all the plot categories this event qualifies for
828  // (for example: charged currents, muon neutrinos, ...)
829  for (icarus::trigger::PlotSandbox const& plotSet: plotSets) {
830 
831  //
832  // overview plots from different thresholds
833  //
834 
835  HistGetter const get { plotSet };
836 
837  // simple efficiency
838  get.Eff("Eff"s).Fill(fired, minCount);
839 
840  // trigger time (if any)
841  if (fired) {
842  get.Hist2D("TriggerTick"s).Fill(minCount, triggerInfo.atTick().value());
843  }
844 
845  //
846  // plots depending on the trigger response
847  // (but not caring of the trigger definition details)
848  //
849 
850  // efficiency plots
851  // (including event plots in the triggered or non-triggered category)
852  helper().fillAllEfficiencyPlots
853  (eventInfo, PMTinfo, triggerInfo, plotSet.demandSandbox(minCountStr));
854 
855  //
856  // add here further trigger-specific plots
857  //
858 
859  } // for all qualifying plot categories
860 
861  } // for all requirements
862 
863  /*
864  * Now fill the plots independent of the trigger response:
865  * the same value is plotted in all plot sets.
866  * (again for all pertinent event categories, e.g. charged currents, etc.)
867  */
868  for (PlotSandbox const& plotSet: plotSets) {
869 
870  //
871  // general plots, independent of trigger definition details
872  //
873  fillEventPlots(eventInfo, plotSet);
874 
875  //
876  // general plots, independent of trigger definition but dependent on
877  // threshold
878  //
879  fillPMTplots(PMTinfo, plotSet);
880 
881  //
882  // trigger-definition specific plots
883  //
884  HistGetter const get(plotSet);
885 
886  // number of primitives
887  get.Hist("NPrimitives"s).Fill(maxPrimitives.second);
888 
889  } // for
890 
891 } // icarus::trigger::MajorityTriggerEfficiencyPlots::plotResponses()
