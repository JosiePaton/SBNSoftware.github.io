35 /**
36  * @brief Applies sliding window patterns to discriminated waveforms.
37  *
38  * This algorithm takes as input one trigger gate for each window.
39  * Each window is identified by an index, and the input gates are assigned one
40  * per window, in order.
41  *
42  * The window mapping that defines the topology of the windows is passed to the
43  * algorithm.
44  *
45  * If a beam gate is present, it is applied to all input before simulating the
46  * pattern. Otherwise, the full input is used.
47  *
48  * For the definition of the windows, see `icarus::trigger::WindowChannelMap`.
49  *
50  */

166  /**
167  * @brief Returns the trigger response for the specified window pattern.
168  * @param windowInfo the topology of the windows
169  * @param pattern the trigger requirement pattern
170  * @param gates trigger gates, one per window
171  * @return a `TriggerInfo_t` record with the response of the pattern
172  *
173  * The input `gates` represent all the trigger gates relevant to the trigger
174  * determination, in a list by window index: the first gate belongs to the
175  * window `0`, the second one to the window `1` and so on.
176  * The topology `windowInfo` identifies a window and its topology, i.e. its
177  * neighbours, by window index. It is required and assumed that the values of
178  * the window indices match the ones in the `gates` list.
179  * This function applies the requirements in the specified `pattern` to the
180  * window identified by `windowInfo`, using the actual trigger gates from
181  * the `gates` list as needed.
182  *
183  * The return value includes the first tick at which the requirements have
184  * been all met.
185  * Note that if a beam gate needs to be applied, it should be applies to
186  * all `gates` before calling this function (see e.g. `applyBeamGate()`).
187  *
188  */
189  TriggerInfo_t applyWindowPattern(
190  WindowTopology_t::WindowInfo_t const& windowInfo,
191  WindowPattern_t const& pattern,
192  TriggerGates_t const& gates
